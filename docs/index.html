TraceBot

Local DevOps & Automated Testing Agent

Track 2: Localized AI Agents Team: NeoDev AMD Radeon + ROCm

What is TraceBot?

TraceBot is an autonomous multi-agent system that monitors your local Git repositories, identifies test coverage gaps, generates unit tests using a locally-running LLM, and self-corrects failures through an intelligent debugging loop.

It runs entirely on local AMD Radeon hardware using the ROCm software stack with Ollama for inference — no cloud APIs, no data leaves your machine.

How It Works

1
Monitor — Detects changed Python files in your Git repository
2
Analyze — Parses code via AST to find functions without test coverage
3
Generate — Prompts Llama 3 (local) to write comprehensive unittest files
4
Debug — Runs tests; if they fail, feeds tracebacks to the LLM for self-correction (up to 3 iterations)
5
Report — Summarizes results via the FastAPI dashboard

Architecture

tracebot/ ├── main.py # FastAPI server + endpoints ├── config.py # Environment-based configuration ├── agents/ │ └── coordinator.py # Pipeline: Analyze → Generate → Debug → Report ├── tools/ │ ├── git_monitor.py # Git change detection │ ├── code_parser.py # AST-based function extraction │ ├── test_runner.py # unittest subprocess execution │ └── file_ops.py # Safe file I/O └── requirements.txt

API Endpoints

POST/run

Trigger a new analysis + test generation run

GET/run/{run_id}

Check the status of a specific run

GET/runs

List all runs and their results

GET/health

Health check — model status and configuration

Quick Start

# 1. Install dependencies pip install -r tracebot/requirements.txt # 2. Install and start Ollama with Llama 3 curl -fsSL https://ollama.com/install.sh | sh ollama pull llama3 # 3. Run TraceBot cd tracebot && python main.py # 4. Trigger a run curl -X POST http://localhost:8000/run \ -H "Content-Type: application/json" \ -d '{"repo_path": "/path/to/your/repo"}'
